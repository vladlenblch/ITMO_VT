## Функционал веб-приложения
- Пользователь вводит координаты цели (X/Y/Z, минимум 500 м) и сразу видит стартовую точку и цель на 3D-графике.
- Перед расчётом задаются скорость и направление ветра; кнопка «Рассчитать траекторию» отправляет данные по WebSocket, сервер ищет оптимальный вектор запуска и может вернуть кэшированный результат для тех же условий.
- Во время симуляции ветер можно менять на лету: ракета не перенастраивает тягу, поэтому хорошо видно, как порывы уводят её от цели.
- В реальном времени приходят координаты ракеты и цели, скорость ракеты, текущая и минимальная дистанция, момент минимального сближения и факт попадания/промаха либо раннего завершения.
- Визуализация в браузере (Plotly): траектории, стрелка ветра с головой и крыльями, индикаторы высоты/скорости/дистанции; отдельное окно показывает параметры расчёта и применённые формулы.

## Параметры ракеты по умолчанию
- Масса пустая: 60 кг
- Топливо: 48 кг
- Время горения: 12 с
- Тяга: 8000 Н
- Cd: 0.2, площадь: 0.05 м², широта: 40.7°

## Физика
- Интегратор: метод Рунге–Кутта 4-го порядка по шагу `dt`; направление тяги фиксируется в момент старта.
- Тяга (`F_thrust`): постоянная величина вдоль выбранного вектора запуска, работает пока горит топливо.
- Гравитация (`F_gravity`): сила тяжести `m * g`, тянет вниз.
- Аэродинамическое сопротивление (`F_drag`): зависит от относительной скорости `v_rel = v_missile - v_wind`, коэффициента `Cd`, площади `A` и плотности воздуха `rho`; направлено против `v_rel`.
- Кориолис (`F_coriolis`): зависит от широты.
- Топливо: линейный расход, масса не опускается ниже пустой.
- Цель: равномерное движение `r(t) = r0 + v * t`.
- Попадание: расстояние ≤ 5 м; «пролёт» фиксируется при проходе через точку минимального сближения.
